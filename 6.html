There is legislation specific to each field of activity where IT is involved, but two major topics are general: Copyright and GDPR. I have treated them in the "Legal Constraint" page. General Review

3.1 Review the benefits and drawbacks of IT tools and systems used in terms of productivity and efficiency – could you have been more efficient when making your website? How? Could you have done it in teams?

I used the “www.freelogodesign.org” website to make the logo, and for manipulating photos or other graphics I chose to use Microsoft Paint (old, unreliable, but for superficial changes it still does its job ).

Visual Studio Code is so far (but I'm just the debutant in the field) the best code editing tool. It covers a very wide range of programming languages. Allows "debug" (unfortunately very necessary for me). Unfortunately, it does not allow team work or different terminals. Documents worked by different individuals or from different locations need to be transported and associated through other programs.

VSC has an advantage: it can be associated with Github. And so (together) they become an essential tool for code writing. Here I used the Github for the first time. Allows free site publishing and content storage. A weak point of GitHub is that it does not allow VSC work directly on the server. Any change made involves uploading and / or offline testing.

About IT tools and systems used so far, I spoke at the previous points. I have already said that this project can be done in the team (by distributing each of the activities from Kanban) but that it is not ethical (the main purpose of the project is to demonstrate personal qualities).

3.2 Describe ways to improve productivity and efficiency

Productivity and efficiency can be enhanced by better knowledge of tools, languages, and IT programs in use. Gaining experience leads to shorter production times and superior product quality.

3.3 Develop solutions to improve own productivity using IT in digital projects – what would you do differently next time? Team work? Use of Slack?

The next project will go much better and faster. My main mistake was to give priority to content, to the detriment of form and appearance.

3.4 Describe how you would go about testing digital solutions

For this project we had three testing stages. Each of them assumed a “Test - Problem - Solving - Test” cycle until the time I was relatively satisfied with my solution.

The first two tests were performed offline to verify the HTML, CSS, and Javascript scripts.

The last test was made after publishing the website and was made using several browsers and several types of devices (static and mobile).

Follows the continuous cycle “Test - Problem - Solution" (specific to the maintenance process, which involves continuous reporting to feedback).
